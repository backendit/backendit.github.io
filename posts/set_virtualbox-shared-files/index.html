VirtualBox安装Ubuntu增强功能包及设置客户机与宿主机之间文件共享

“因为我们自己使用的系统大多数是Windows或MacOS，所以个人学习Linux一般都会在虚拟机中安装Linux系统的各个发行版，如CentOS，Ubuntu等。这里主要是总结一下在MacOS下的VirtualBox中如何安装Ubuntu系统以及踩过的一些坑和解决办法。 ”

准备工作

  1. 下载Ubuntu系统，桌面版或Server版本(Server版本体积小无桌面)；
  2. 下载VirtualBox虚拟机软件。

整个安装过程就不增加篇章说明了，因为都是在网上都可以找到的，或者自己下一步下一步就知道怎么安装，开始从创建一个Ubuntu虚拟机开始。

安装增强功能

首先在Virtualbox虚拟机的菜单设备中，点击安装增强功能，这时虚拟机设置中存储，就会出现一个VBoxGuestAdditions.iso镜像文件。这个就是VirtualBox的增强功能包。

接着进入Ubuntu系统，使用命令挂载这个增强包。执行命令行

$ sudo mount /dev/cdrom /media/cdrom

挂载以上的iso文件。即将iso镜像挂载到了/media/cdrom目录下(可能会没有/media/cdrom，没有就新建一个)。

这时进入/media/cdrom目录中去，就会看到增强功能包的一些文件了。

然后执行下面的命令来安装。

$ sudo sh VBoxLinuxAdditions.run

最后reboot重启系统即可。

问题及解决

1.遇到问题：在VirtualBox安装增强包时出现错误提示。

原来是虚拟机Ubuntu系统中已经安装了增强镜像了，你在点击安装就会出现这个错误了。所以就不用再去点安装了，或者先把系统中的光驱弹出来。

如果你的Ubuntu是桌面版的，有界面的，直接在界面上选择光驱，然后点击弹出即可。如果没有界面，比如安装的Server版本，体验无界面的原生操作，那就在命令行中使用命令弹出光驱。 弹出光驱执行命令

$ eject

弹出光驱后，重新点击安装增强功能即可。

2.挂载出现问题，如下提示： warning device write-protected mounted read-only 原因可能是磁盘做了写保护。所以解决办法就是临时将磁盘的写保护给关闭了。

sudo hdparm -r0 /dev/sda

然后再次挂载就成功了。

设置文件共享

把宿主机中的文件上传到虚拟机，如果是远程不同IP虚拟机，使用scp命令、ftp工具等都可上传，但是本机学习用的虚拟机上貌似行不通。然后知道共享文件夹，只需要在虚拟机中设置本地的共享文件夹，然后在Linux上挂载到指定的文件夹即可实现双向互传文件了，非常方便。

在虚拟机设置中设置共享文件夹，如设置共享文件夹名称为UbuntuVMShare。

首先在虚拟机Ubuntu中创建一个共享文件目录，也就是挂载点，将Mac上的文件夹挂载。在Mac目录中新建一个共享目录。

在Ubuntu下新建一个文件夹：

$ sudo mkdir /home/MacVMShared
 # 挂载宿主机中的共享文件夹：
 sudo mount -t vboxsf UbuntuVMShare /home/MacVMShared
$ sudo mount -t vboxsf 共享文件夹名称（在设置页面设置的） 挂载的目录

然后进入/home/MacVMShared，就可以看到放在Mac中UbuntuVMShared目录的文件了。当在其中一端中增加或删除文件都能实现同步(共享)了。非常实用。
